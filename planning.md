## Tools

### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
This tool searches the listings dataset for clothing items that match the user's description, size, and budget. Returns matching listings sorted by relevance.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): Keywords describing the item the user wants to find.
- `size` (str): Desired clothing size to filter results
- `max_price` (float): Maximum amount the user is willing to spend.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of listing dictionaries containing fields such as id, title, description, style_tags, category, size, condition, price, colors, brand, and platform.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
The agent will inform the user that no matching listings were found and suggest broadening the search or increase the max budget. The planning loop stops and doesn't call additional tools.

---

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
This tool uses the selected listing and the user's wardrobe to generate one or more outfit suggestions. The tool explains how the new item can be styled with existing wardrobe pieces.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The selected clothing item returned from search_listings.
- `wardrobe` (dict): The user's wardrobe containing clothing and accessory items.

**What it returns:**
<!-- Describe the return value -->
A string containing one or more complete outfit recommendations and styling suggestions.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe is empty, the tool generates general styling advice based on the new item instead of referencing wardrobe pieces.

---

### Tool 3: create_fit_card

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
This tool generates a short social-media-style caption based on the selected item and outfit recommendation. The caption should feel natural and shareable.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `outfit` (str): The outfit recommendation generated by suggest_outfit.
- `new_item` (dict): The selected clothing item being featured.

**What it returns:**
<!-- Describe the return value -->
A short caption string describing the outfit and highlighting the thrifted item.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the outfit data is incomplete? -->
If the outfit text is missing or incomplete, the tool returns a descriptive error message instead of generating a fit card.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->
How does your agent decide which tool to call next?

1. Receive the user's clothing request.
2. Call search_listings(description, size, max_price).
3. Check if the returned results list is empty.
4. If no results are found, store an error message in the session and return immediately.
5. If results exist, save the first result as selected_item.
6. Call suggest_outfit(selected_item, wardrobe).
7. Save the returned outfit recommendation as outfit_suggestion.
8. Call create_fit_card(outfit_suggestion, selected_item).
9. Save the generated caption as fit_card.
10. Return the completed session containing the selected item, outfit suggestion, and fit card.

---

## State Management

**How does information from one tool get passed to the next?**
<!-- Describe how your agent stores and accesses state within a session. What data is tracked? How is it passed between tool calls? -->
The agent stores information in a session dictionary throughout the interaction. After search_listings runs, the selected listing is stored as session["selected_item"]. The selected item is passed into suggest_outfit, and the resulting recommendation is stored as session["outfit_suggestion"]. That recommendation and the selected item are then passed into create_fit_card, and the final caption is stored as session["fit_card"]. Any errors are stored in session["error"].

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Tell the user no matching listings were found, suggest adjusting size, description, or budget, and stop the workflow. |
| suggest_outfit | Wardrobe is empty | Generate general styling advice for the selected item without using wardrobe data. |
| create_fit_card | Outfit input is missing or incomplete | Return a descriptive error message explaining that outfit information is required before creating a fit card. |
