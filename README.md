## The Mock Listings Dataset

`data/listings.json` contains 40 mock secondhand listings across categories (tops, bottoms, outerwear, shoes, accessories) and styles (vintage, y2k, grunge, cottagecore, streetwear, and more).

Each listing has: `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

Load it with:
```python

```

## Tool Inventory

### `search_listings(description: str, size: str | None = None, max_price: float | None = None) -> list[dict]`

**Purpose:**
Searches the mock secondhand listings dataset and returns items that match the user's description, optional size, and optional price limit.

**Inputs:**

* `description` (str): Keywords describing the desired item.
* `size` (str | None): Optional clothing size filter.
* `max_price` (float | None): Optional maximum price filter.

**Returns:**

* `list[dict]`: A list of matching listing dictionaries sorted by relevance.

---

### `suggest_outfit(new_item: dict, wardrobe: dict) -> str`

**Purpose:**
Generates outfit recommendations using a selected thrifted item and the user's wardrobe.

**Inputs:**

* `new_item` (dict): Listing selected from the search results.
* `wardrobe` (dict): User wardrobe containing clothing items.

**Returns:**

* `str`: One or more outfit recommendations generated by the LLM.

---

### `create_fit_card(outfit: str, new_item: dict) -> str`

**Purpose:**
Creates a social-media-style caption based on the outfit recommendation and selected item.

**Inputs:**

* `outfit` (str): Outfit recommendation generated by `suggest_outfit()`.
* `new_item` (dict): Selected thrifted item.

**Returns:**

* `str`: A short caption suitable for a social media post.

---

## Interaction Walkthrough

**User query:**

`vintage graphic tee under $30`

### Step 1 — Tool called

* **Tool:** `search_listings`
* **Input:** `description="vintage graphic tee"`, `size=None`, `max_price=30`
* **Why this tool:** The planning loop first searches the listings dataset for matching items.
* **Output:** A list of matching listings. The top result is selected and stored in `session["selected_item"]`.

### Step 2 — Tool called

* **Tool:** `suggest_outfit`
* **Input:** `selected_item`, `wardrobe`
* **Why this tool:** After selecting the best matching listing, the agent generates outfit recommendations using the user's wardrobe.
* **Output:** A styling recommendation stored in `session["outfit_suggestion"]`.

### Step 3 — Tool called

* **Tool:** `create_fit_card`
* **Input:** `outfit_suggestion`, `selected_item`
* **Why this tool:** The final step generates a shareable caption using the outfit recommendation and thrifted item.
* **Output:** A social-media-style caption stored in `session["fit_card"]`.

### Final output to user

The user receives:

1. The top thrift listing found.
2. An outfit recommendation using the selected item.
3. A social-media-style fit card caption.

## Error Handling and Fail Points

| Tool              | Failure mode                                               | Agent response                                                                       |
| ----------------- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| `search_listings` | No listings match the user's description, size, or budget. | Sets `session["error"]` and stops the planning loop before calling additional tools. |
| `suggest_outfit`  | User wardrobe is empty.                                    | Generates general styling advice instead of referencing wardrobe items.              |
| `create_fit_card` | Outfit string is empty or missing.                         | Returns an informative error message rather than raising an exception.               |

## Planning Loop Explanation

The FitFindr agent follows a sequential planning loop. First, the user's natural language query is parsed to extract a description, optional size, and optional maximum price. The agent then calls `search_listings()` using those parsed values.

If no matching listings are found, the agent stores an error message in `session["error"]` and terminates early without calling any additional tools. If results are found, the first result is stored as `session["selected_item"]` and passed to `suggest_outfit()` along with the selected wardrobe.

The outfit recommendation returned by `suggest_outfit()` is stored in `session["outfit_suggestion"]` and then passed to `create_fit_card()`. The generated caption is stored in `session["fit_card"]`. Once all steps are complete, the final session dictionary is returned and displayed through the Gradio interface.

## State Management Approach

The agent uses a session dictionary as the single source of truth for each interaction. The session stores the original query, parsed query parameters, search results, selected item, wardrobe, outfit recommendation, fit card, and any error message.

After `search_listings()` runs, the top result is stored in `session["selected_item"]`. This value is passed directly into `suggest_outfit()`. The resulting recommendation is stored in `session["outfit_suggestion"]`, which is then passed into `create_fit_card()`. The final caption is stored in `session["fit_card"]`. This approach allows information to be passed between tools without recomputing earlier results.
